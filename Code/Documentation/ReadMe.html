24 Plug-In Template 3.0

This ReadMe file contains some important information about 24 Plug-In Template, such as system requirements, installation guide, quick start, troubleshooting, license notice, contact information, and copyright statement.

1. Introduction

24 Plug-In Template (formerly named 24U FM Template) is template project for developing third-party FileMaker external function plug-ins. It is intended for beginning plug-in developers as well as for experienced developers who don't want to spend time configuring parts of the FileMaker Plug-In SDK, but want to concentrate on developing new functionality for FileMaker Pro, FileMaker Pro Advanced, and FileMaker Server.

The following development environments are supported in a form of ready to use project files by this version of 24U Plug-In Template:

  • XCode 2.4 and better on Mac OS X 10.4 Tiger
  • XCode 3.0 and better on Mac OS X 10.5 Leopard
  • XCode 3.2 and better on Mac OS X 10.6 Snow Leopard
  • Microsoft Visual Studio .NET 2003 on Windows XP SP3
  • Microsoft Visual Studio 2010 Express or better on Windows XP SP3, Windows Vista, Windows 7

If you have a different version of one of these environments you may need to adjust settings to be able to build plug-ins.

2. System requirements

Recommended configuration:

Mac OS X: Any Intel Mac, 4096 MB RAM, Mac OS X 10.6 Snow Leopard, XCode 3.2, FileMaker Pro 11 Advanced.

Windows: 1.6 GHz processor, 4096 MB RAM, Windows 7, Microsoft Visual Studio 2010, FileMaker Pro 11 Advanced.

Minimum requirements:

Mac OS X: Macintosh, 512 MB RAM, Mac OS X 10.4 Tiger, XCode 2.4, FileMaker Pro 8.5.

Windows: 500 MHz processor, 512 MB RAM, Windows XP SP3, Microsoft Visual Studio 2003, FileMaker Pro 8.5.

3. Installation and testing build

To create a testing plug-in build of 24 Plug-In Template, follow these steps:

  1. If you did not do this yet, download and expand the 24 Plug-In Template distribution archive.
  2. The 24U Plug-In Template folder contains all necessary files to build the plug-in in all supported environments. Copy this folder to the location with read-write permissions.
  3. The Projects folder of the 24U Plug-In Template contains all supported environments. Choose and open the project file.
  4. Try to build the project in the IDE. It should end with no error. If some error occurs please try to look to the troubleshooting section of this readme.
  5. The Output folder inside the 24U Plug-In Template now contains either FMPluginDebug.fmplugin or FMPluginDebug.fmx. This file should be installed to the FileMaker Pro. The easiest way to do it is to copy the file to the folder Extensions which is inside the FileMaker Pro installation folder.
  6. Start or restart the FileMaker Pro. After the start go to the Preferences / Options in the menu and choose Plug-In. There should be a "My Plug-In". If the plug-in is not there please check all previous steps.
  7. The plug-in is successfully installed now. To check if it is working properly try to open the file "Template Version.fp7" from the Examples folder inside the 24U Plug-In Template. Call the version script and the plug-in should return its version.

4. Troubleshooting

Plug-In is not visible in the FileMaker Pro
Check if the plug-in binary is present in the one of "Extensions" folders of the FileMaker Pro. FileMaker Pro reads plug-ins from the folder named Extensions which can be placed in:

  • [User home folder]/Library/Application Support/FileMaker/Extensions on Mac OS X or [User home folder]\Local Settings\Application Data\FileMaker\Extensions on Windows. This is the preferred location for plug-ins in FileMaker Pro 9 and newer.
  • /Applications/FileMaker Pro 8.5 Advanced/Extensions on Mac OS X or C:\Program Files\FileMaker\FileMaker Pro 8.5 Advanced\Extensions on Windows. This location is preferred for FileMaker Pro 8.5 and older. You usually need administration privileges to install the plug-in into this location on both operating systems.

I've got "object file compiled with -mlong-branch which is no longer needed" warning when compiling in XCode 3.2
This warning is caused by Mac OS X 10.4u SDK which was installed with XCode 3.2. This is probably the problem of the XCode and it should be fixed with some newer version of the XCode. This does not affect plug-ins functionality.
If you want to get rid of this warning please read Apple mailing list.
